> 확정 데이터: [item_design.md](item_design.md)

## 아이템 종류

| 종류 | 설명 | 드롭 대상 | 상태 |
|------|------|-----------|------|
| 장비 | 영웅에 장착하는 장비 | 정예, 보스 | 설계 중 |
| 카드 | 캐릭터 슬롯에 장착 → 스킬 발동 용도 | 모든 몬스터 (극악 확률) | 설계 중 |
| 강화 재료 | 장비 강화/크래프팅 소모품 (A/B 2종) | 모든 몬스터 | 설계 중 |
| 포션 | 체력 회복 및 특수효과 | 미정 | 미기획 |

## 카드 시스템

### 기본 구조
- 모든 몬스터는 고유한 카드 1종 보유 (48종 = 16베이스 × 3)
- 몬스터 처치 시 극악의 확률로 카드 드롭 (장기 파밍 목표)
- 카드 = **스킬 해금 열쇠**: 1장 획득 시 해당 몬스터의 고유 스킬 해금
- 카드는 **거래 가능** 아이템 (거래로 얻은 카드도 스킬 해금 가능)
- 카드 자체가 캐릭터 특성 레이어 → 코스트 시스템에 영향 없음

### 카드의 이중 역할

| 역할 | 설명 | 조건 |
|------|------|------|
| **스킬 해금** | 카드 획득 → 해당 스킬 등록, 스킬슬롯에 장착 가능 | 카드 1장 보유 |
| **도감 등록** | 직접 드롭 획득 시 도감 기록 → 패시브 보너스 자동 적용 | **직접 드롭만** (거래 획득 불가) |

### 스킬 슬롯 (4슬롯, 점진적 해금)

| 해금 레벨 | 스킬슬롯 수 |
|----------|------------|
| Lv1 | 1슬롯 |
| Lv5 | 2슬롯 |
| Lv15 | 3슬롯 |
| Lv30 | 4슬롯 |

- 해금된 스킬 중 슬롯 수만큼 장착 가능
- 장착된 스킬: **적중 시 n% 확률로 자동 발동** (액티브 스킬, 자동전투)
- 슬롯 점진적 해금 → 초반 허전함 방지 + 레벨업 보람

### 카드 레벨업 (스킬 강화)
중복 카드를 소모하여 스킬 레벨 상승. SP 시스템 없음.

| 스킬 레벨 | 누적 카드 | 효과 |
|----------|----------|------|
| Lv1 | 1장 | 스킬 해금 (기본 효과) |
| Lv2 | +N장 | 발동률/위력 상승 |
| Lv3 | +N장 | 발동률/위력 상승 |
| ... | ... | ... |
| 만렙 | 총 N장 | 스킬 진화 + 추가 효과 |

> 구체적 레벨/장수는 밸런싱 시 확정
> 중복 카드 = 항상 가치 있음 (거래 없어도 스킬 강화 재료)

### 도감 시스템
- **직접 드롭 획득 시에만** 도감 등록 (거래로 얻은 카드는 도감 불변)
- 도감 보너스: 보유만으로 자동 적용 (슬롯 장착 불필요)
- 도감 보너스 내용: 개별 카드 보너스 or 세트 완성 보너스 (상세 미정)

### 카드 드롭 후 흐름
```
몬스터 처치 → 카드 드롭 → 첫 장: 스킬 해금 + 도감 등록
                        → 중복: 스킬 레벨업 재료로 소모
                        → 잉여: 거래소 판매 (추후)
```

## 아이템 등급 체계

| 등급 | 획득처 | 접사 구성 | 세트포인트 | 특징 |
|------|--------|-----------|-----------|------|
| 매직 (Magic) | 필드 | 접두 OR 접미 1개 | 1 | 크래프팅 베이스. 코스트 낮음 |
| 레어 (Rare) | 필드/보스 | 접두 + 접미 2개 | 2 | 드롭 완성품. 코스트 중간 |
| 크래프트 (Craft) | 제작 | 접두 + 접미 2개 | 2 | 죄종 선택 가능 + 전용 옵션. 코스트 높음 |
| 유니크 (Unique) | 보스 전용 | 고정 1개 | 1 | 고정 효과 + 수치 범위 1개. 코스트 최고 |

> 일반 등급 제거: 강화 재료 드롭으로 대체. 필드에서 드롭되는 모든 아이템이 의미를 가짐
> 유니크: 완전 랜덤이 아닌 고정 효과 + 수치 범위 1개 → 희소성과 예측 가능성 동시 확보

### 등급별 역할 분담

- **매직**: 크래프팅 베이스 탐색 / 저비용 세트포인트 확보 수단
- **레어**: 메인 파밍 대상. 운이 좋으면 최고의 조합 / 코스트 대비 세트포인트 효율 좋음
- **크래프트**: 원하는 죄종 조합을 설계. 레어로 운에 맡기지 않고 통제하는 루트
- **유니크**: 고유 효과로 빌드 방향을 정의. 코스트가 높아 전 슬롯 유니크는 불가능

## 타겟 파밍 구조 (2축)

유저의 파밍 의사결정을 두 축으로 분리.

| 유저의 질문 | 결정 축 | 결정 요소 |
|------------|---------|-----------|
| "어떤 죄종 접사가 필요하지?" | **지역(필드)** | 필드마다 특정 죄종 접사 확률 편향 |
| "어떤 장비 부위가 필요하지?" | **몬스터 베이스** | 몬스터 종족별 장비 부위 드롭 가중치 |

### 지역 → 죄종 편향
- 각 필드는 7죄종 중 하나의 테마를 가짐
- 해당 필드에서 장비 드롭 시, 테마 죄종의 접사가 높은 확률로 부여됨
- **편향이지 독점은 아님**: 테마 죄종 확률 상승, 나머지도 낮은 확률로 등장

### 몬스터 베이스 → 부위 편향
- 몬스터 종족에 따라 특정 장비 부위 드롭률 상승
- `monster_drop_equipment.csv`에서 부위별 가중치 관리
- 유저가 체감할 수 있는 수준의 편향 필요 (균등 배분 지양)

### 몬스터 타입은 드롭에 영향 없음
- 타입(Normal/Demon/Undead)은 공통 접사 옵션 `vs_[type]_damage` 대상 분류 역할만 담당
- 드롭 편향과 전투 속성 역할을 분리하여 시스템 복잡도 관리

## 아이템 드랍 로직

```
1. 몬스터 처치
   └─ 정보: level, field-level, field-sin, monster-base, grade, spawn-grade

2. 드랍 여부 결정
   └─ spawn-grade별 판정 횟수 (일반: 1회, 정예: 3회)

3. 아이템 종류 결정
   └─ grade로 드랍율 결정 (gold, equip, card, material 등)

4. (장비) 장비 부위 결정
   └─ monster-base별 부위 가중치 참조 (타겟 파밍 축 2)

5. 베이스 품질 결정
   └─ item-base + 몬스터 점수 → 베타분포

6. 아이템 등급 결정
   └─ 몬스터 점수 기반 랜덤
   └─ 유니크: 특정 구역의 특정 몬스터만 드랍

7. 접두/접미사 결정
   └─ field-sin 기반 확률 편향 (타겟 파밍 축 1)
   └─ 매직: 1개, 레어: 2개, 유니크: 고정

8. 옵션 개수 결정
   └─ 매직: 1~3개, 레어: 2~6개, 유니크: 고정

9. 옵션 수치 결정
   └─ 몬스터 점수 기반 베타분포

10. 최종 아이템 생성
```
