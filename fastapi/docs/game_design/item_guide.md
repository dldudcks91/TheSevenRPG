## 아이템 종류

| 종류 | 설명 | 드롭 대상 | 상태 |
|------|------|-----------|------|
| 골드 | 범용 화폐 | 모든 몬스터 | 소모처 설계 중 |
| 장비 | 무기/갑옷/투구/장갑/신발 | 정예, 보스 | 설계 중 |
| 카드 | 몬스터별 고유 1종, 도감 등록 → 스킬 해금 | 모든 몬스터 (극악 확률) | 설계 중 |
| 포션 | 체력 회복 / 버프 | 모든 몬스터 | 미기획 |
| 광석 | 크래프팅 범용 재료 (등급 계층형) | 모든 몬스터 | 미기획 |
| 낙인 | 7죄종별 1종, 크래프팅 죄종 선택 재료 | 챕터 보스 (희귀) | 미기획 |
| 퀘스트 재료 | NPC 납품 / 특정 교환 | 몬스터별 고유 | 미기획 |

## 카드 시스템 ✅ 확정

### 기본 구조
```
카드 드롭 (랜덤 수치 범주)
  ├→ 도감 자동 등록 (첫 획득 시 스킬 해금)
  ├→ 카드 전용 인벤토리 탭에 아이템으로 보관
  └→ 슬롯 장착 / 카드 레벨업 재료 / 분해 / 거래소 판매
```

- 모든 몬스터는 고유한 카드 1종 보유 (48종 = 16베이스 × 3)
- 몬스터 처치 시 극악의 확률로 카드 드롭
- 카드는 **인벤토리 아이템** (카드 전용 탭, 장비 인벤과 별도)
- 카드 자체가 캐릭터 특성 레이어 → 코스트 시스템에 영향 없음

### 카드 수치 (범주)
- 드롭 시 수치가 랜덤 범주 내에서 결정 (발동확률, 효과수치)
- 같은 카드라도 수치가 다를 수 있음
- 더 좋은 수치 카드를 파밍하여 교체 가능

### 도감
- 카드 첫 획득 시 도감 자동 등록 + **스킬 해금**
- 도감 레벨업 시스템 없음 (수집 기록 역할만)
- 카드를 팔거나 분해해도 도감 등록은 유지
- 도감 그룹 보너스: ⬜ 재설계 필요

### 스킬 슬롯 (4슬롯, 점진적 해금)

| 해금 레벨 | 스킬슬롯 수 |
|----------|------------|
| Lv1 | 1슬롯 |
| Lv5 | 2슬롯 |
| Lv15 | 3슬롯 |
| Lv30 | 4슬롯 |

- 같은 카드는 1개만 장착 가능
- 장착된 스킬: 트리거 조건 충족 시 자동 발동 (자동전투)
- 상세 트리거 7종 → [battle_guide.md](battle_guide.md)

### 카드 레벨업
- 재료: **같은 카드 N장 + 카드 영혼 N개**
- 레벨업 시 드롭 시 결정된 수치 유지, 추가 보너스 획득
- 필요 수량 및 보너스 수치는 밸런싱 단계에서 확정

### 카드 영혼
- 카드 분해 시 획득 (카드 종류 무관, 범용 재료)
- 거래소 거래 가능 여부 추후 결정

### 거래
- 카드 아이템 거래소 거래 가능 (추후 도입)
- 수치 좋은 카드가 고가, 나쁜 수치 카드는 분해 or 저가 판매

## 아이템 등급 체계

| 등급 | 획득처 | 접사 구성 | 세트포인트 | 특징 |
|------|--------|-----------|-----------|------|
| 매직 (Magic) | 필드 | 죄종1 + 공통1 | 1 | 크래프팅 베이스. 코스트 낮음 |
| 레어 (Rare) | 필드/보스 | 죄종2 + 공통2 | 2 | 드롭 완성품. 코스트 중간 |
| 크래프트 (Craft) | 제작 | 죄종2 + 공통2 | 2 | 죄종 선택 가능 + 전용 옵션. 코스트 높음 |
| 유니크 (Unique) | 보스 전용 | 고정 1개 | 1 | 고정 효과 + 수치 범위 1개. 코스트 최고 |

> 일반 등급 제거: 강화 재료 드롭으로 대체. 필드에서 드롭되는 모든 아이템이 의미를 가짐
> 유니크: 완전 랜덤이 아닌 고정 효과 + 수치 범위 1개 → 희소성과 예측 가능성 동시 확보

### 등급별 역할 분담

- **매직**: 크래프팅 베이스 탐색 / 저비용 세트포인트 확보 수단
- **레어**: 메인 파밍 대상. 운이 좋으면 최고의 조합 / 코스트 대비 세트포인트 효율 좋음
- **크래프트**: 원하는 죄종 조합을 설계. 레어로 운에 맡기지 않고 통제하는 루트
- **유니크**: 고유 효과로 빌드 방향을 정의. 코스트가 높아 전 슬롯 유니크는 불가능

## 타겟 파밍 구조 (3축)

유저의 파밍 의사결정을 세 축으로 분리.

| 유저의 질문 | 결정 축 | 결정 요소 |
|------------|---------|-----------|
| "어떤 죄종 접사가 필요하지?" | **지역(필드)** | 필드마다 특정 죄종 접사 확률 편향 |
| "어떤 장비 부위가 필요하지?" | **몬스터 베이스** | 몬스터 종족별 장비 부위 드롭 가중치 |
| "크래프팅에 어떤 죄종을 쓰지?" | **챕터 보스** | 챕터 보스별 해당 죄종 낙인 드롭 |

### 지역 → 죄종 편향
- 각 필드는 7죄종 중 하나의 테마를 가짐
- 해당 필드에서 장비 드롭 시, 테마 죄종의 접사가 높은 확률로 부여됨
- **편향이지 독점은 아님**: 테마 죄종 확률 상승, 나머지도 낮은 확률로 등장

### 몬스터 베이스 → 부위 편향
- 몬스터 종족에 따라 특정 장비 부위 드롭률 상승
- `monster_drop_equipment.csv`에서 부위별 가중치 관리
- 유저가 체감할 수 있는 수준의 편향 필요 (균등 배분 지양)

### 몬스터 타입은 드롭에 영향 없음
- 타입(Normal/Demon/Undead)은 공통 접사 옵션 `vs_[type]_damage` 대상 분류 역할만 담당
- 드롭 편향과 전투 속성 역할을 분리하여 시스템 복잡도 관리

## 아이템 드랍 로직

```
1. 몬스터 처치
   └─ 정보: level, field-level, field-sin, monster-base, grade, spawn-grade

2. 드랍 여부 결정
   └─ spawn-grade별 판정 횟수 (일반: 1회, 정예: 3회)

3. 아이템 종류 결정
   └─ grade로 드랍율 결정 (gold, equip, card, material 등)

4. (장비) 장비 부위 결정
   └─ monster-base별 부위 가중치 참조 (타겟 파밍 축 2)

5. 베이스 품질 결정
   └─ item-base + 몬스터 점수 → 베타분포

6. 아이템 등급 결정
   └─ 몬스터 점수 기반 랜덤
   └─ 유니크: 특정 구역의 특정 몬스터만 드랍

7. 접두/접미사 결정 (죄종 옵션)
   └─ field-sin 기반 확률 편향 (타겟 파밍 축 1)
   └─ 매직: 1개, 레어/크래프트: 2개, 유니크: 고정

8. 공통 옵션 결정
   └─ 부위별 공통 옵션 풀에서 랜덤 선택
   └─ 매직: 1개, 레어/크래프트: 2개, 유니크: 고정
   └─ 무기는 기본옵션(공격력%), 갑옷은 기본옵션(방어력%)이 추가로 항상 부여

9. 옵션 수치 결정
   └─ 몬스터 점수 기반 베타분포

10. 최종 아이템 생성
```
