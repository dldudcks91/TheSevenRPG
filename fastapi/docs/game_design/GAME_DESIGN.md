## 1. 게임 개요

- **장르**: 웹 RPG (아이템 파밍 중심)
- **테마**: 7죄종 + 중세신화
- **핵심 재미**: 아이템 파밍과 장착에 대한 밸런싱
- **컨셉**: "전투는 간단하게, 아이템 파밍을 주력으로"
- **핵심 구조**: 전투 **전** 준비(아이템·세트·스탯)가 전투 결과를 결정

## 2. 핵심 게임 루프

### 월드 구조
- **스테이지 방식**: 던전 보스 클리어 → 다음 스테이지 해금
- 각 스테이지는 **필드**와 **던전**으로 구성

### 필드
- 지역 클릭 → 입장 → 몹 **연속 자동 스폰** (방치 파밍)
- 중간중간 **엘리트(정예) 몬스터** 등장 → 드랍 판정 3회
- 드랍: 일반 / 매직 등급 아이템
- 사망 시: 필드 밖으로 이동, **획득 아이템 유지**

### 던전
- 지역 클릭 → 입장 → **일반 4마리 + 정예 1마리** 세트를 3~4회 반복 → **던전 보스** 등장
- 1:1 자동전투로 순서대로 처치
- 보스 클리어 → 다음 스테이지 해금
- 사망 시: 던전 밖으로 이동, **획득 아이템 유지**

### 스폰 등급별 드롭 규칙

| 스폰 등급 | 등장 장소 | 장비 드롭 | 기타 드롭 |
|-----------|-----------|-----------|-----------|
| 일반 | 필드, 던전 | X | 골드, 강화 재료, 카드(극악 확률) |
| 정예 | 필드(중간), 던전(세트당 1마리) | O (매직/레어) | 골드, 강화 재료, 카드 |
| 보스 | 던전 마지막 | O (레어/유니크) | 골드, 강화 재료 |

### 핵심 루프
```
필드 파밍 (일반/매직 아이템)
    ↓
아이템 장착 및 빌드 최적화
    ↓
던전 도전 → 보스 클리어 (레어/유니크 아이템)
    ↓
다음 스테이지 해금 → 반복
```

## 3. 전투 시스템

### 기본 전투 흐름
- **1:1 완전 자동전투** (플레이어 개입 없음)
- 공격속도 쿨타임마다 자동으로 상대 공격
- HP가 0이 되면 패배
- 전투에는 애니메이션 포함

### 카드 스킬 발동
- 장비 소켓에 카드를 장착하면 전투 중 **조건 충족 시 자동 발동**

| 발동 조건 | 설명 |
|-----------|------|
| 공격 시 n% | 내가 공격 액션을 취할 때 |
| 타격(명중) 시 n% | 내 공격이 실제로 명중했을 때 |
| 피격 시 n% | 내가 공격을 맞았을 때 |

### 상태이상

| 상태이상 | 효과 | 카운터 대상 |
|---------|------|------------|
| 스턴 | 일정 시간 공격 불가 | 고공속 적 |
| 빙결 | 공격속도 감소 | 공속형 빌드 |
| 중독 | 매 초 고정 데미지 | 탱커 |
| 화상 | 체력회복 감소 | 흡혈(색욕 세트) |

## 4. 스탯 시스템

### 기본 스탯

- **최대 레벨**: 99
- **레벨업 시 스탯 포인트**: 5개
- **총 스탯 포인트**: 98 × 5 = **490개**
- **기본 HP**: 100 (레벨업 시 자동 증가 없음 → 체력 스탯 투자로만 증가)

| 스탯 | 영문 | 1포인트 효과 | 490 올인 시 |
|------|------|-------------|------------|
| 힘 | Strength | 공격력 +0.5% | 무기 기본공격력 × 3.45배 |
| 민첩 | Dexterity | 공격속도 +0.3%, 명중률 +0.5% | 공격속도 2.47배, 명중률 3.45배 |
| 체력 | Life | 최대HP +10 (고정) | 100 + 4,900 = **5,000 HP** |
| 운 | Luck | 치명타확률 +0.1%, 치명타데미지 +0.3%, 회피율 +0.1% | 치명타확률 +49%, 치명타데미지 +147%, 회피율 +49% |
| 코스트 | Cost | 최대코스트 +2 | 레벨당 자동 +1, 스탯 투자 시 +2/포인트 |

### 세부 스탯 공식

| 세부 스탯 | 공식 |
|-----------|------|
| 최대HP | `(100 + 체력×10) × (1 + Σ아이템HP%)` |
| 공격력 | `무기기본공격력 × (1 + 힘×0.005) × (1 + Σ아이템공격력%)` |
| 공격속도 | `무기기본공격속도 × (1 + 민첩×0.003) × (1 + Σ아이템공속%)` |
| 명중률 | `(1 + 민첩×0.005) × (1 + Σ아이템명중%)` |
| 치명타확률 | `운×0.001 + Σ아이템치확` |
| 치명타데미지 | `1.5 + 운×0.003 + Σ아이템치뎀` |
| 회피율 | `운×0.001 + Σ아이템회피` |
| 방어력 감소율 | `방어력 / (방어력 + 100)` |
| 마법저항력 감소율 | `마법저항력 / (마법저항력 + 100)` |

> 계수(0.005 등)는 전투 시뮬레이션 후 튜닝 예정

## 5. 코스트 시스템

장비 착용에 코스트 한도가 필요하며, 좋은 장비일수록 더 많은 코스트를 소비합니다.

### 플레이어 최대코스트
```
최대코스트 = (레벨 × 1) + (코스트스탯 × 2)

예시)
레벨 99, 코스트스탯   0 → 99 코스트
레벨 99, 코스트스탯  50 → 199 코스트
레벨 99, 코스트스탯 100 → 299 코스트
```

### 장비 코스트 공식
```
장비 코스트 = 장비레벨 × 옵션레벨 × 등급배율
```

**등급배율**

| 등급 | 배율 |
|------|------|
| 매직 | × 1.0 |
| 레어 | × 1.5 |
| 크래프트 | × 2.0 |
| 유니크 | × 2.5 |

> 일반 등급 제거됨 (강화 재료로 대체)
> 배율 수치는 시뮬레이션 후 튜닝 예정

> **장비레벨**은 플레이어에게 숨겨지며, 레벨에 따라 뜰 수 있는 옵션 종류가 달라짐
> **크래프트 코스트가 레어보다 높은 이유**: 죄종 카테고리 선택권(통제권)에 대한 프리미엄

## 6. 7죄종 시스템

각 죄종은 장비 세트의 테마이자, 접두/접미사 옵션의 방향성을 결정합니다.

| 영문 | 한국어 | 특성 방향 | 설명 |
|------|--------|-----------|------|
| Wrath | 분노 | 강력한 공격 특화 | 주체할 수 없는 분노의 감정, 복수와 증오 |
| Sloth | 나태 | 강력한 옵션 + 패널티(-옵션) 특화 | 흥미 부재, 습관적 무기력 |
| Lust | 색욕 | 아군 강화 (버프·흡혈) | 돈·힘·성에 대한 강렬한 열망 |
| Envy | 시기 | 상대 능력 약화·탈취 (디버프) | 타인의 소유·특성에 대한 분노와 갈망 |
| Pride | 오만 | 우월함 과시 | 인간을 신처럼 만들어주는 능력에 대한 도착 |
| Gluttony | 폭식 | 스택형 스탯 특화 | 지나친 탐닉과 과소비 |
| Greed | 탐욕 | 확률·운 기반 도박 | 탐욕스러운 물질 소유 추구 |

## 7. 아이템 시스템

### 아이템 종류

| 종류 | 설명 | 드롭 대상 | 상태 |
|------|------|-----------|------|
| 장비 | 영웅에 장착하는 장비 | 정예, 보스 | 설계 중 |
| 카드 | 캐릭터 슬롯에 장착 → 스킬 발동 용도 | 모든 몬스터 (극악 확률) | 설계 중 |
| 강화 재료 | 장비 강화/크래프팅 소모품 (A/B 2종) | 모든 몬스터 | 설계 중 |
| 흔적 | 죄종별 파밍 효율 버프 아이템 | 모든 몬스터 (죄종 지역 편향) | 1차 확정 |
| 포션 | 체력 회복 및 특수효과 | 미정 | 미기획 |

### 카드 시스템

#### 기본 구조
- 모든 몬스터는 고유한 카드 1종 보유
- 몬스터 처치 시 극악의 확률로 카드 드랍 (장기 파밍 목표)
- 카드는 **캐릭터 전용 슬롯 4개**에만 장착 가능 (장비 소켓 방식 아님)
- 카드 자체가 캐릭터 특성 레이어 → 코스트 시스템에 영향 없음

#### 각인 레벨 시스템
같은 카드를 중복 획득할수록 각인 레벨 상승.

| 각인 레벨 | 필요 장수 | 효과 |
|-----------|----------|------|
| Lv1 | 1장 | 기본 옵션 활성화 |
| Lv2 | 3장 | 옵션 수치 강화 |
| Lv3 | 10장 | 최대 효과 + 추가 효과 언락 |

> 중복 카드를 합산하는 방식 (인벤 관리 간소화)

#### 카드 옵션 방향
- **아이템에서 얻을 수 없는 고유 옵션**: 경험치 획득량, 골드 드롭률, 특정 속성 저항, 부활 등
- 카드가 장비를 보완하는 역할 → 장비와 카드 풀이 서로 겹치지 않음

#### 활성화 슬롯
- **4슬롯 고정**: 보유한 카드 중 4종만 동시 효과 발동
- "어떤 카드 4개를 선택할 것인가" 자체가 빌드 요소

### 흔적 시스템

#### 핵심 개념
> "죄종 지역을 파밍할수록 그 죄종의 흔적이 쌓이고, 쌓인 흔적이 파밍 효율을 높인다."

- **전투력과 완전히 분리**: 흔적은 파밍 효율만 올림. 전투 스탯 영향 없음
- **역할**: 타겟 파밍 3축 (1축: 지역→죄종접사, 2축: 몬스터베이스→부위, **3축: 흔적→파밍목적**)

#### 기본 구조
- **7종류** (죄종당 1종), 게이지 방식 (0~100)
- **드롭 아이템** 형태: 획득 시 즉시 해당 죄종 게이지 +수치 (수치는 아이템마다 다름)
- **소모**: 몬스터 처치 시 자동 소모
- **순환**: 해당 지역 파밍 중 → 드롭으로 상쇄 ≈ 유지 / 다른 지역 이동 → 자연 감소
- **동시 활성화**: 최대 2~3종 제한 ("어떤 흔적 조합을 유지할까"가 빌드 요소)
- **활성화 조건**: 게이지 일정량 이상 보유 시 효과 발동

#### 죄종별 효과 (파밍 효율 특화)

| 죄종 | 흔적 효과 | 파밍 목적 |
|------|-----------|-----------|
| 분노 | 정예 몬스터 스폰 빈도 ↑ | 장비 드롭 기회 극대화 |
| 나태 | 드롭 수량 절반, 등급 최소 레어 보장 | 소수 고품질 파밍 |
| 색욕 | 카드 드롭 확률 ↑ | 카드 수집 |
| 시기 | 현재 지역 죄종 접사 편향 2배 | 특정 접사 타겟 파밍 |
| 오만 | 경험치 획득량 ↑ | 레벨업 |
| 폭식 | 강화 재료 드롭량 ↑ | 크래프팅 재료 수집 |
| 탐욕 | 골드 드롭 ↑ | 재화 파밍 |

#### 미확정 수치
- 게이지 소모량 (처치당 몇씩 감소)
- 드롭 아이템 수치 범위 (min~max)
- 동시 활성화 슬롯 수 (2 or 3)
- 효과 발동 임계값 (게이지 몇 이상일 때)
- 효과 수치 (정예 스폰 몇 % 증가 등)

### 아이템 등급

| 등급 | 획득처 | 접사 구성 | 세트포인트 | 특징 |
|------|--------|-----------|-----------|------|
| 매직 (Magic) | 필드 | 접두 OR 접미 1개 | 1 | 크래프팅 베이스. 코스트 낮음 |
| 레어 (Rare) | 필드/보스 | 접두 + 접미 2개 | 2 | 드롭 완성품. 코스트 중간 |
| 크래프트 (Craft) | 제작 | 접두 + 접미 2개 | 2 | 죄종 선택 가능 + 전용 옵션. 코스트 높음 |
| 유니크 (Unique) | 보스 전용 | 고정 1개 | 1 | 고정 효과 + 수치 범위 1개. 코스트 최고 |

> **일반 등급 제거**: 강화 재료 드롭으로 대체. 필드에서 드롭되는 모든 아이템이 의미를 가짐
> **유니크 능력치**: 완전 랜덤이 아닌 고정 효과 + 수치 범위 1개 → 희소성과 예측 가능성 동시 확보

### 등급별 역할 분담

- **매직**: 크래프팅 베이스 탐색 / 저비용 세트포인트 확보 수단
- **레어**: 메인 파밍 대상. 운이 좋으면 최고의 조합 / 코스트 대비 세트포인트 효율 좋음
- **크래프트**: 원하는 죄종 조합을 설계. 레어로 운에 맡기지 않고 통제하는 루트
- **유니크**: 고유 효과로 빌드 방향을 정의. 코스트가 높아 전 슬롯 유니크는 불가능

### 파밍 흐름

```
일반몹 처치 (필드/던전 공통)
  ├─ 골드             → 재화 축적
  ├─ 강화 재료 A/B    → 장비 강화, 크래프팅 소모
  └─ 카드 (극악 확률)  → 장기 수집 목표

정예몹 처치 (필드/던전 공통)
  ├─ 위 일반몹 드롭 +
  ├─ 매직 장비        → 좋은 베이스면 크래프팅, 아니면 분해
  └─ 레어 장비        → 장착 검토 / 불필요하면 분해

던전 보스 처치
  ├─ 레어 장비        → 고품질 완성품
  └─ 유니크 장비      → 빌드 핵심 슬롯
```

### 타겟 파밍 구조 (2축)

유저의 파밍 의사결정을 두 축으로 분리한다.

| 유저의 질문 | 결정 축 | 결정 요소 |
|------------|---------|-----------|
| "어떤 죄종 접사가 필요하지?" | **지역(필드)** | 필드마다 특정 죄종 접사 확률 편향 |
| "어떤 장비 부위가 필요하지?" | **몬스터 베이스** | 몬스터 종족별 장비 부위 드롭 가중치 |

#### 지역 → 죄종 편향
- 각 필드는 7죄종 중 하나의 테마를 가짐 (예: "분노의 숲", "나태의 늪")
- 해당 필드에서 장비 드롭 시, 테마 죄종의 접사가 높은 확률로 부여됨
- **편향이지 독점은 아님**: 테마 죄종 확률 상승, 나머지도 낮은 확률로 등장
- 한 스테이지 안에 여러 필드가 존재하여, 초반부터 복수의 죄종에 접근 가능

#### 몬스터 베이스 → 부위 편향
- 몬스터 종족(Humanoid, Beast, Skeleton 등)에 따라 특정 장비 부위 드롭률 상승
- `monster_drop_equipment.csv`에서 부위별 가중치 관리
- 유저가 체감할 수 있는 수준의 편향 필요 (균등 배분 지양)

#### 몬스터 타입(Normal/Demon/Undead)은 드롭에 영향 없음
- 몬스터 타입은 공통 접사 옵션(vs_Normal/vs_Demon/vs_Undead 피해)의 대상 분류 역할만 담당
- 드롭 편향과 전투 속성 역할을 분리하여 시스템 복잡도 관리

### 아이템 드랍 로직

```
1. 몬스터 처치
   └─ 정보: level, field-level, field-sin(지역 죄종), monster-base, grade, spawn-grade(일반/정예)
   └─ → monster-score 계산 (공식 미정)

2. 드랍 여부 결정
   └─ spawn-grade별 판정 횟수 (일반: 1회, 정예: 3회)

3. 아이템 종류 결정
   └─ grade로 드랍율 결정
   └─ 종류: gold, equip, card, material 등 (구상 중)

4. (장비) 장비 부위 결정
   └─ monster-base별 부위 가중치 참조 (타겟 파밍 축 2)

5. 베이스 품질 결정
   └─ item-base 정보 + 몬스터 점수 → 베타분포로 결정

6. 아이템 등급 결정
   └─ 몬스터 점수 기반 랜덤
   └─ 일반 등급 제거 → 해당 구간은 강화 재료 드랍으로 대체
   └─ 유니크: 특정 구역의 특정 몬스터만 드랍

7. 접두/접미사 결정
   └─ field-sin(지역 죄종) 기반 확률 편향 적용 (타겟 파밍 축 1)
   └─ 매직: 접두 OR 접미 1개, 레어: 접두 + 접미 2개, 유니크: 고정

8. 옵션 개수 결정
   └─ 매직: 1~3개, 레어: 2~6개, 유니크: 고정

9. 옵션 수치 결정
   └─ 몬스터 점수 기반 베타분포

10. 최종 아이템 생성
```

## 9. 크래프팅 시스템

### 개념

매직 아이템(접두 OR 접미 하나)을 베이스로, 비어있는 쪽의 **죄종 카테고리를 직접 선택**해서 접사를 추가하는 시스템.

```
매직 아이템 [접두: 분노] 보유
  + 재료 소모 (아이템 레벨에 비례)
  + 접미사 죄종 선택: [오만]
  → 크래프트 아이템 [접두: 분노] + [접미: 오만의 랜덤 옵션]
```

### 핵심 규칙

- **베이스 매직 옵션 유지**: 기존 접두/접미는 변경되지 않음
- **죄종 카테고리 선택**: 7죄종 중 하나 선택 → 해당 카테고리 내 옵션은 랜덤
- **재료 비용**: 아이템 레벨에 비례. 고레벨 크래프팅일수록 재료 많이 소모
- **결과물 코스트**: 레어보다 높음 (통제권에 대한 프리미엄)
- **크래프트 전용 옵션**: 레어/유니크에서 얻을 수 없는 옵션 존재 (세부 미정)

### 레어 vs 크래프트 비교

| | 레어 | 크래프트 |
|---|---|---|
| 획득 방법 | 드롭 | 제작 |
| 접두사 | 완전 RNG | 매직 베이스에서 선택 가능 |
| 접미사 | 완전 RNG | 죄종 카테고리 선택 (옵션은 RNG) |
| 코스트 | 낮음 | 높음 |
| 전용 옵션 | 없음 | 있음 (세부 미정) |
| 세트포인트 | 2 | 2 |

> 레어: 저렴하지만 두 옵션 모두 운에 맡김
> 크래프트: 비싸지만 한쪽 죄종을 내가 선택

### 크래프팅 재료

- **획득처**: 필드 드롭 (강화 재료와 구분 필요 여부 검토 중)
- **소모량**: 아이템 레벨 기반 공식 (수치 미정)

## 10. 낙인 시스템

> D2의 주얼에 대응하는 소켓 삽입형 아이템. 죄종 타입 매칭으로 차별화.

### 핵심 개념

- **낙인(Brand)**: 장비 소켓에 삽입하는 아이템. 완전 랜덤 스탯 부여.
- **타입 매칭**: 낙인의 죄종 타입과 장비의 접두/접미 죄종이 일치하는 소켓에만 삽입 가능.
  - 예: `분노의 낙인` → [분노] 접두사를 가진 장비의 소켓 1번에만 삽입 가능
- **역할 분리**: 낙인은 죄종 스탯을 직접 제공하지 않음. 장비 접사와 역할 중복 방지.

### 소켓 구조

| 소켓 | 부여 조건 | 낙인 타입 |
|------|----------|---------|
| 소켓 1 (기본) | 모든 장비에 기본 보유 | 접두 죄종 낙인 전용 |
| 소켓 2 (랜덤) | 드롭 시 랜덤 옵션으로 2번째 소켓 확정 | 접미 죄종 낙인 전용 |

- 2소켓 발생: 드롭 시 랜덤 옵션 중 하나로 확정 (별도 제작/강화 불필요)
- 접두와 접미의 죄종이 다를 수 있음 → 소켓 1, 2에 서로 다른 죄종 낙인 삽입 가능

### 낙인 스탯

- **완전 랜덤**: 낙인의 옵션은 죄종과 무관하게 랜덤 부여
- 구체적인 옵션 풀은 밸런싱 단계에서 확정 (현재 보류)
- 방향: 장비 접사와 겹치지 않는 옵션 (카드처럼 장비 보완 역할)

### 미확정 사항

- 낙인 옵션 풀 (밸런싱 단계로 보류)
- 낙인 드롭처 (죄종 지역 편향 여부)
- 낙인 등급 유무
- 코스트 시스템과 낙인의 관계

## 11. 파밍 레이어 교차 구조

> 각 파밍 레이어가 서로 독립적으로 동작하는 "병렬 구조"의 한계를 극복하기 위한 설계 방향.
> 레이어 A의 완성이 레이어 B의 전제조건이 되는 "교차 구조"를 목표로 한다.

### 교차 체인 1 — 던전 티어 → 접사 티어 (1순위, 핵심)

```
던전 티어 1~3  → 매직 장비만 드롭 (접두 OR 접미 1개)
던전 티어 4~6  → 레어 드롭 가능 (접두 + 접미)
던전 티어 7~9  → 레어 + 접사 수치 상위 티어 가능
던전 티어 10+  → 크래프트/유니크 드롭 가능
```

- 상위 던전 클리어 = 상위 접사의 전제조건
- 크래프팅 베이스 자체가 상위 던전에서만 드롭 → 하위에서 크래프팅 시도 불가
- PoE의 iLvl 구조와 동일한 역할

### 교차 체인 2 — 낙인 분해 → 죄종 파편 (2순위)

```
쓸모없는 낙인 분해 → 죄종 전용 파편 획득
죄종 전용 파편     → 해당 죄종 접사 고정 크래프팅 가능
                     (탐욕 파편 → 크래프팅 시 탐욕 접사 1개 확정)
```

- D2 큐브 조합과 동일한 역할
- 낙인이 쓰레기면 파편으로 전환 → 낙인 파밍이 크래프팅에도 기여

### 교차 체인 3 — 카드 각인 Lv3 → 세트 포인트 +1 (3순위)

```
특정 죄종 몬스터 카드 각인 Lv3 달성
    ↓
해당 카드 장착 시 → 해당 죄종 세트 포인트 +1 추가 인정
    ↓
세트 레벨이 평소보다 1 높아짐 → 세트 보너스 강화
```

- 카드 파밍 이유: "Lv3 각인이 세트 레벨에 직접 기여"
- "어떤 카드 4개를 선택할 것인가"가 세트 빌드와 연동됨

### 교차 체인 4 — 세트 레벨 × 낙인 (피로도 낮은 교차)

```
죄종 세트 레벨 X 달성 (안정적 조건 - 한 번 달성하면 유지)
    ↓
해당 죄종 낙인의 추가 효과 활성화
```

- 흔적 게이지 × 낙인 직접 연동은 **기각** (관리 피로도 과다)
- 세트 레벨은 게이지처럼 자동 감소하지 않음 → 달성 후 안정적 유지

### 교차 구조 우선순위

| 순위 | 교차 체인 | 설계 난이도 | 임팩트 |
|-----|----------|------------|--------|
| 1 | 던전 티어 → 접사 티어 | 낮음 (드롭 테이블 수정) | 높음 |
| 2 | 낙인 분해 → 죄종 파편 | 낮음 (분해 레시피 추가) | 중간 |
| 3 | 카드 각인 Lv3 → 세트+1 | 중간 (세트 계산 로직 수정) | 높음 |
| 4 | 세트 레벨 → 낙인 효과 활성화 | 중간 | 중간 |

## 12. 세트 보너스 시스템

### 세트포인트 구조

장비의 접두/접미사 각각이 해당 죄종에 세트포인트 1씩 기여.

| 등급 | 세트포인트/슬롯 | 비고 |
|------|---------------|------|
| 매직 | 1 | 접두 OR 접미 하나 |
| 레어 | 2 | 접두 + 접미 각각 1 |
| 크래프트 | 2 | 접두 + 접미 각각 1 |
| 유니크 | 1 | 고정 접사 하나 |

- **5 슬롯** 기준 최대 세트포인트: 레어/크래프트 풀착용 시 **10포인트**
- **죄종 레벨 상한**: 7레벨 (초기 구현은 5레벨)
- **빌드 예시**: `3분노 + 4교만` = 7포인트 소모 (10 중 7 사용)
- **유니크 선택 시 트레이드**: 해당 슬롯 세트포인트 -1 + 코스트 증가 → 고유 효과 획득

### 빌드 전략 갈래

```
7레벨 극대화 빌드: 한 죄종 7포인트 집중 → 각성 효과 확보 + 나머지 3포인트 분배
4+3 분산 빌드:    두 죄종의 중간 단계 보너스를 동시 활용 (각성 없음)
유니크 활용 빌드: 유니크 1~2개 + 세트포인트 희생 → 고유 효과와 세트 조합
```

> 각성(6레벨) 보너스의 강도가 "집중 vs 분산" 메타를 결정하는 핵심 변수

죄종별 장비를 모을수록 레벨별 보너스 획득 (1~7레벨, 초기 구현 1~5레벨)

## 14. 몬스터 시스템

### 몬스터 타입 (3종)

| 타입 | 영문 | 설명 | 하위 base 예시 |
|------|------|------|---------------|
| 일반 | Normal | 자연 생물 및 피조물 | Wolf, Golem, Bear, Spider... |
| 데몬 | Demon | 지옥/마법 계열 존재 | Imp, Succubus, Hellhound... |
| 언데드 | Undead | 죽어서 되살아난 존재 | Skeleton, Zombie, Ghost, Vampire... |

- 헌터 아이템 옵션: `vs_Normal / vs_Demon / vs_Undead 피해 +%`
- 타입은 장비 공통 옵션의 대상 분류로만 사용 (전투 속성 시스템 없음)

### 몬스터 데이터 구조

| 레이어 | 컬럼 | 역할 |
|--------|------|------|
| monster_type | Normal / Demon / Undead | 헌터 아이템 대상 분류 |
| monster_base | Wolf / Golem / Imp / Skeleton... | 장비 부위 드롭 분포 결정 |
| monster_grade | 1~4 | 드롭 테이블 품질 결정 |

- **monster_type** → 공통 접사 옵션 `vs_[type]_damage` 대상
- **monster_base** → `monster_drop_equipment.csv`에서 부위별 드롭 가중치 참조
- **monster_grade** → `monster_drop_config.csv`에서 드롭 종류별 확률 참조
