# 미정의 원칙 총정리

> 기획서 전체를 교차 검증하여 도출한 **정의되지 않았지만 구현/밸런스에 직접적 영향을 미치는 원칙** 목록.
> 각 항목은 논의 후 확정하고, 확정 시 관련 문서에 반영 + 이 문서에서 ✅ 처리한다.

## 우선순위 요약

### 즉시 확정 필요 (문서 간 모순 → 구현 시 혼선)
- ✅ L-1: 레벨업 자동 성장 없음
- ✅ L-2: 도감 레벨업 시스템 폐기
- ✅ L-3: 7챕터 × (일반 3스테이지 + 보스 1) = 28스테이지
- ✅ L-4~6: 상태이상 효과 status_effect.csv 기준으로 통일

### 밸런싱 전 확정 필요 (수치 시뮬레이션 불가)
- ✅ A-1~4: 사망 패널티 확정
- ✅ C-1~4: 상태이상 발동확률 + 지속시간 + 중첩 규칙 확정
- ✅ B-3~4 + 신규: 경직 발동 조건, FHR 공식, 쿨타임 처리 확정
- ✅ F-3: 접두/접미 동일 죄종 중복 불가
- ✅ F-4: 폭식 패널티 수치 확정 (밸런싱 시 세부 조정)

### 구현 전 확정 필요 (코드 분기에 영향)
- ✅ E-1~4: 코스트 시스템 전체 확정
- ✅ F-1: 유니크 고정 접사, 해당 죄종 세트포인트 기여
- ✅ F-2: 베이직 죄종 마을에서만 변경
- ✅ F-5: 세트 상한 6, 초과 없음
- ✅ G-1 (재정의): D2 구조 채용 (dlvl→mlvl→ilvl). **dlvl 스테이지 배분 수치는 3안 미확정** ← 이슈
- ✅ G-2: ilvl = 드롭 몬스터의 mlvl. mlvl 보정 체계 확정. 드롭 등급 하드 제한 폐기 → mlvl 기반 확률
- ✅ G-3: 각 보스 전용 유니크 드롭
- ✅ G-5: 장비 인벤토리 초기 20슬롯
- ✅ H-1~5: 웨이브 구조 전체 확정
- I-1: 스킬 동시 발동 규칙

## M. 레벨업 동기 설계 — 레벨을 목표로 만드는 구조 (논의 중)

> **문제 정의**: 현재 레벨업 보상이 스탯 5pt(체감 미미)뿐이라 유저가 레벨업을 목표하지 않음.
> 장비가 성장의 압도적 비중을 차지하고, 레벨은 보조축에 불과.
> 아래 3가지 장치를 조합하여 "모든 레벨에 목표가 있는" 구조를 만드는 방안 논의 중.

### M-1. 장비 요구 레벨

| 항목 | 내용 | 상태 |
|------|------|------|
| **개념** | 장비 착용에 캐릭터 레벨 제한. 요구 레벨 미달 시 장착 불가 | ✅ equipment_guide에 추가 완료 |
| **공식** | `요구 레벨 = item_level + 등급보정 (매직+0, 레어+2, 유니크+5)` | ✅ 확정 |
| **역할** | "좋은 장비가 떨어졌지만 못 낀다" → 레벨업 갈망 유발 (불규칙 목표) | |
| **코스트와의 분리** | 요구 레벨 = "언제 낄 수 있는가" (수직), 코스트 = "뭘 낄 것인가" (수평) | |

### M-2. 스탯 브레이크포인트 (논의 중)

| 항목 | 내용 | 상태 |
|------|------|------|
| **개념** | 스탯이 특정 수치(10/20/30)에 도달하면 상시 패시브 특성 해금 | ⬜ 논의 중 |
| **구조** | 5종 스탯 × 3단계 = 15개 패시브 특성 | ⬜ 특성 목록 미설계 |
| **역할** | 매 레벨 "다음 브레이크포인트까지 Npt" = 상시 마이크로 목표 | |
| **카드와의 분리** | 카드 = 전투 중 확률 발동 액티브, 브레이크포인트 = 상시 패시브 → 역할 충돌 없음 | |
| **빌드 영향** | "힘20 관통 먼저 vs 체력20 생존 먼저" = 초반부터 의미 있는 스탯 배분 고민 | |
| **미확정** | 5종 × 3단계 특성 내용, 수치 밸런싱, 스탯 리셋 시 특성 회수 방식 | ⬜ |

**특성 초안 (예시, 미확정)**:
```
힘 10/20/30:   [강타] 5% 확률 ×1.5뎀 / [관통] 방어 10% 무시 / [분쇄] 경직 +0.3초
민첩 10/20/30: [속사] 공속 +5% / [잔상] 회피 시 반격 / [질풍] 공속 +15%
체력 10/20/30: [강인] 경직저항 +15% / [생존본능] HP30%↓ DEF+30% / [불굴] 1회 부활
운 10/20/30:   [행운] 드롭률 +5% / [직감] 치명타뎀 +20% / [기적] 극저확률 즉사
코스트 10/20/30: [경량화] 코스트 -5% / [효율] 세트포인트 +1 / [초월] 코스트 +20
```

### M-3. 레벨 마일스톤 해금 (논의 중)

| 항목 | 내용 | 상태 |
|------|------|------|
| **개념** | 특정 레벨 도달 시 기능/보상 해금. 3~5레벨 간격으로 "뭔가 열리는" 리듬 | ⬜ 논의 중 |
| **역할** | "다음 레벨에 뭐가 열리지?" = 고정 간격 목표 | |
| **문제점** | 줄 수 있는 보상이 부족. 대부분 미구현 시스템(분해/포션/재련)이라 Phase 의존 | ⬜ |

**배치 초안 (미확정)**:
```
Lv3:  장비 분해 해금 (Phase 21 선행)
Lv5:  스킬슬롯 2번 (기존)
Lv7:  카드 1장 확정 지급 (신규)
Lv10: 마일스톤 보상 + 보너스 스탯 (신규)
Lv12: 포션 사용 해금 (Phase 18 선행)
Lv15: 스킬슬롯 3번 (기존)
Lv20: PVP 해금 (기존) + 마일스톤 보상
Lv30: 스킬슬롯 4번 (기존)
Lv40: 그라인드 구간 + 마일스톤 보상
Lv50: 만렙 보상
```

### 세 장치의 역할 분담 (종합)

| 장치 | 목표 유형 | 주기 | 체감 |
|------|----------|------|------|
| 장비 요구 레벨 | "이 장비 끼려면 Lv12!" | 장비 드롭 시마다 (불규칙) | 갈망 |
| 스탯 브레이크포인트 | "힘 20까지 2pt!" | 매 2~3레벨 (상시) | 기대 |
| 레벨 마일스톤 | "Lv10에 보상!" | 3~5레벨 간격 (고정) | 기대 |

> **결론**: 세 장치가 합쳐지면 빈 레벨이 없어지는 구조. M-1은 확정, M-2/M-3은 세부 설계 후 확정 예정.
