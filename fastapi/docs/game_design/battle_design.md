## 상태이상 7종

> 수치 데이터: `fastapi/meta_data/status_effect.csv`

각 챕터의 죄종에 대응하는 상태이상을 부여한다. 빌드 간 상성 구조를 만드는 핵심 요소.

## 스탯별 수치

> 수치 데이터: `fastapi/meta_data/stat_config.csv`

스탯은 5종 (힘/민첩/체력/운/코스트). 레벨업마다 포인트를 자유 배분하며 리셋 비용(골드)이 발생한다.

## 레벨 구간 설계

| 구간 | 레벨 | 방식 | 목표 시간 | 비고 |
|------|------|------|----------|------|
| 1챕터 | Lv1 → Lv10 | 스토리 진행 | 1시간 | |
| 2챕터 | Lv10 → Lv20 | 스토리 진행 | 2시간 | |
| 3챕터 | Lv20 → Lv30 | 스토리 진행 | 4시간 | |
| 4챕터 | Lv30 → Lv40 | 스토리 진행 | 6시간 | 현재 구현 범위 끝 |
| 그라인드 구간 | Lv40 → Lv50 | 파밍 노가다 | 미정 | 하드코어 파밍 핵심 구간 |

- Ch1~Ch4 스토리 합계: **13시간** (Lv1→40)
- 5~7챕터는 Lv40 이상 구간으로 추후 배치
- Lv40 이후 레벨업이 매우 어렵게 설계 → 레벨 1 차이가 PvP에서 체감됨

## 스테이지 구조 ✅확정

### 스테이지 구성

> 등급별 배율·전투시간 데이터: `fastapi/meta_data/spawn_grade_config.csv`

- 일반스테이지: 일반 9 + 정예 3 + 스테이지보스 1 = **13마리**
- 보스스테이지: 일반 9 + 정예 1 + 스테이지보스 1 + 챕터보스 1 = **12마리**
- 챕터당 **4스테이지** (일반 3 + 보스 1) / 챕터 클리어 = 4스테이지 각 1회 클리어
- 동레벨 적정장비 기준 1챕터 클리어: 약 **24분** (일반 3런 + 보스 1런)

## 경험치 설계 ✅확정

### 필요 XP 커브

> 레벨별 필요 XP 전체 테이블: `fastapi/meta_data/level_exp_table.csv`

- **기준 XP** (Lv1→2): **10**
- **스토리 구간** (Lv1~40): 레벨당 **×1.3** 증가
- **그라인드 구간** (Lv40~50): 레벨당 **×2.0** 증가
- **몬스터 XP 증가율**: 몬스터 레벨당 **×1.2** / 등급 비율: 일반 1x / 정예 3x / 스테이지보스 10x / 챕터보스 20x
- Ch1~Ch4 런당 XP: 일반스테이지 28단위, 보스스테이지 42단위 / 챕터당 약 ×2.2배 증가

## 코스트 등급배율

> 수치 데이터: `fastapi/meta_data/equip_rarity_config.csv` (base_cost 컬럼)

일반 등급 제거됨 (강화 재료로 대체). 배율 수치는 시뮬레이션 후 튜닝 예정.
