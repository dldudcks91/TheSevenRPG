# 세계 구조 & 챕터 가이드

> 월드 구성 원칙, 스테이지 구조, 챕터 설계 규칙
> 확정 데이터: [chapter_design.md](chapter_design.md)

---

## 전체 구조

```
7챕터 × 3스테이지 = 21스테이지
각 챕터 = 스테이지 1~3 + 챕터 보스 (관문)
스테이지당 예상 소요: 2~5일
챕터 클리어 예상: 10~20일
전체 클리어 예상: 70~140일
```

## 스테이지 구성 원칙

- 3스테이지 클리어 → **챕터 보스** 등장 (관문)
- 챕터 보스 클리어 → 다음 챕터 해금
- 스테이지 보스 21마리 + 챕터 보스 7마리 = **보스 28종**
- **스테이지 단일 타입**: 1스테이지 = 1몬스터 타입 (Normal/Demon/Undead)
- 챕터마다 스테이지 타입 순서를 다르게 설정 → 패턴 고정 방지
- 각 챕터의 상태이상이 해당 지역 테마와 직결 — 환경 자체가 위협

## 스테이지 진행 UI

### 기본 구조 — 횡스크롤 필드 진행

```
[배경: 횡스크롤 필드]

  다음 몬스터     현재 전투           플레이어
  (실루엣)      (1:1 전투 중)
    👤 ← ← ←    🐺 ⚔️ 🧑

─────────────────────────────────────
[  Wave 1/3  ][■■■□ 4/13][드롭 로그]
```

- 처치할 때마다 캐릭터가 앞으로 이동 → 진행 체감
- 다음 몬스터가 실루엣으로 대기 → 정예가 보이면 긴장감
- 하단: 웨이브 번호 + 진행 바(13마리 중 몇 번째) + 드롭 로그

### 일반/정예 등장

```
캐릭터가 앞으로 걸어감
→ 몬스터와 마주침 (멈춤)
→ 1:1 전투 시작 (자동)
→ 처치
→ 드롭 아이템 팝업 (골드 +50, 광석 ×1 등)
→ 다시 앞으로 걸어감
→ 다음 몬스터
```

### 정예 등장

```
다음 몬스터 위치에 정예 실루엣 (빛나는 아우라)
→ 마주침
→ "분노의 고블린 전사" 이름 표시
→ 특성 아이콘 3개 표시 (격분 / 단단한 / 재생하는)
→ 전투 시작
```

### 웨이브 전환

```
4마리 처치 (일반3 + 정예1)
→ "Wave 2" 텍스트 표시 (1초)
→ 배경 약간 변화 또는 스크롤 가속
→ 바로 다음 몬스터 등장 (쉬는 시간 없음)
```

### 보스 등장

```
12마리 처치 완료
→ 화면 어두워짐 + 보스 이름 컷인
→ "아바돈 — 파멸의 수호자"
→ 상단에 대형 HP 바 등장
→ BGM 전환
→ 1:1 전투 시작
```

### 배경 구간

| 구간 | 배경 |
|------|------|
| 웨이브 1 | 스테이지 입구 |
| 웨이브 2 | 스테이지 중반 |
| 웨이브 3 | 스테이지 깊은 곳 |
| 보스 | 보스 전용 배경 (어둡게) |

> 같은 챕터 내에서는 색조 변화로 구간 구분 가능

## 아트 디자인 원칙

### 던전 색채 아이덴티티
- 계절 요소 없음. 모든 챕터는 기본적으로 **어두운 톤**을 유지
- 각 챕터는 **죄악의 색이 오염원으로 깔리는 "빛의 오염" 개념**으로 개성을 표현
- 밝은 색채가 아닌, 어둠 위에 깔리는 특정 색조로 챕터 구분
