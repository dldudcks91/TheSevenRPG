# 몬스터 설계 가이드

> 몬스터 설계 원칙, 베이스 분류, 데이터 구조
> 챕터별 인스턴스 데이터: [monster_design.md](monster_design.md)

---

## 설계 원칙

### 베이스 vs 인스턴스
- **베이스(Base)**: 내부 분류 클래스. 스탯 설계·드롭 가중치·역할의 기준.
- **인스턴스(Instance)**: 플레이어에게 보이는 실제 몬스터. 챕터마다 고유 이름·외형·서사.
- 같은 Skeleton 베이스라도 Ch1의 스켈레톤과 Ch3의 스켈레톤은 **다른 존재**처럼 보여야 한다.

### 챕터 테마 연동
- 각 챕터의 지역 테마·죄악·상태이상이 몬스터 외형에 녹아있어야 한다.
- 예: Ch1 분노(화상) → 불꽃에 그을리고 분노에 물든 형태
- **스테이지 배경 이름도 참고한다** — 몬스터는 그 배경 위에 자연스럽게 존재해야 한다
- 예: "파멸의 진영" → 군사 진영의 악마 병사 / "핏빛 교전지대" → 전투 한복판의 전사 / "원한의 묘지" → 묘지에서 되살아난 언데드

### 전투 기반
- **1:1 자동전투** — 몬스터는 플레이어와 1대1로 싸우는 단일 전투 단위로 설계

### 스테이지 구성
- 1스테이지 = 몬스터 3종(A·B·C) 고정 설계 (유닛 타입은 `GAME_DESIGN.md` 챕터·스테이지별 구성표를 따른다)
- 정예는 **런타임에 랜덤 생성** — 스테이지 내 노말 3종 중 랜덤 유닛 + 랜덤 접두사 조합
- 정예는 미리 정의하지 않는다
- **스테이지 총 13마리**: 노말 9 + 정예 3 + 스테이지 보스 1
- **웨이브 구조**: [노말3 + 정예1] × 3웨이브 + 보스1

### 스폰 순서
- **스토리 모드 (최초 클리어)**: 종별 집중형 — 한 종을 연속 배치하여 순서대로 소개
  - `AAA + Elite(A) → BBB + Elite(B) → CCC + Elite(C) → Boss`
- **파밍 모드 (재입장)**: 완전 랜덤 — 3종 중 랜덤 배치, 정예도 랜덤
  - `랜덤3 + Elite(랜덤) → 랜덤3 + Elite(랜덤) → 랜덤3 + Elite(랜덤) → Boss`

### 네이밍 규칙
- **노말**: `[몬스터 이름] [역할]` — 예: 고블린 전사, 오크 돌격병, 스켈레톤 궁수
- **정예**: `[접두사] [몬스터 이름] [역할]` — 런타임 조합, 사전 정의 없음

### 노말 몬스터 인스턴스 설계 규칙
- **베이스 풀 참조**: 스테이지 구성표의 베이스 풀에서 3종을 선택
- **역할 차별화**: 3종이 서로 다른 전투 역할을 가져야 한다 (예: 근접/원거리/탱커)
- **monster_idx 채번**: `CCSS` 형식 — CC=챕터번호, S=스테이지번호, 순번 1~3
  - 예: Ch1 Stage2 → 1201, 1202, 1203 / Ch3 Stage1 → 3101, 3102, 3103
- **리소스명**: `ch{N}_{base}_{role}` — 예: `ch1_goblin_scout`, `ch2_wolf_stalker`
- **아트 프롬프트 형식**: `16-bit pixel art, [외형 묘사], [챕터 테마 반영], white background, fantasy RPG enemy sprite`

### 스테이지 보스 선정 규칙
- 1스테이지 = 1보스. 챕터당 3스테이지이므로 **챕터당 3보스**
- **타입 일치 필수**: 보스의 monster_type은 해당 스테이지의 타입과 동일해야 한다
  - 예: Demon 스테이지 → Demon 보스, Undead 스테이지 → Undead 보스
- **베이스 매핑 필수**: 보스도 16종 베이스 중 하나에 매핑한다 (드롭·스탯 연동)
- **선정 우선순위**:
  1. 유명세 — 게이머/대중문화에서 이름만으로 알 수 있는가
  2. 죄종 연관도 — 해당 챕터 보스(사탄/레비아탄 등)와 서사적으로 연결되는가
  3. 외형 매칭 — 매핑할 베이스와 원전 외형이 자연스럽게 어울리는가
- **출전 범위**: 성경·신화·민속·문학·고에티아 순으로 유명한 이름 우선
- **중복 금지**: 한 번 사용한 이름은 다른 챕터에서 재사용하지 않는다
- **챕터 보스 제외**: 사탄/레비아탄/맘몬/벨페고르/바알제붑/아스모데우스/루시퍼는 스테이지 보스로 사용하지 않는다
- 후보 목록: `data/스테이지보스_후보_악마목록.md`

## 몬스터 타입 (3종)

| 타입 | 영문 | 설명 |
|------|------|------|
| 일반 | Normal | 자연 생물 및 피조물 |
| 데몬 | Demon | 지옥/마법 계열 존재 (고블린, 오크 포함) |
| 언데드 | Undead | 죽어서 되살아난 존재 |

- 헌터 아이템 옵션: `vs_Normal / vs_Demon / vs_Undead 피해 +%`
- 타입은 장비 공통 옵션의 대상 분류로만 사용 (전투 속성 시스템 없음)
- **스테이지 단일 타입**: 1스테이지 = 1타입만 등장 → vs_type 옵션이 실질적 의미를 가짐

## 몬스터 베이스 (16종)

**챕터 간 베이스 공유 허용** — 같은 베이스라도 챕터마다 이름·스탯·속성을 변형해 재등장 가능.

### Normal (자연 생물·피조물)

| 베이스 | 크기 | 역할 | 외형 특징 |
|--------|------|------|----------|
| Wolf | 소~중형 | 고속 딜러 | 사나운 야생 늑대 |
| Yeti | 중~대형 | 강타·돌격형 | 설산의 거대 설인 |
| Troll | 대형 | 방어 극대화 | 둔중하고 두꺼운 피부의 거인 |
| Lizardman | 소~중형 | 균형형 | 직립 도마뱀 전사·주술사 |
| Golem | 중~대형 | 극단적 방어 | 돌·철로 만들어진 무기물 |
| Human | 중형 | 균형형 전사 | 죄악에 물든 타락한 인간 병사 |

### Demon (지옥·마법 계열)

| 베이스 | 크기 | 역할 | 외형 특징 |
|--------|------|------|----------|
| Imp | 소형 | 빠른 마법형 | 작고 날렵한 소악마 |
| Goblin | 소형 | 잔꾀·집단형 | 무리 지어 다니는 소형 악마 병사 |
| Succubus | 중형 | 디버프·마법형 | 유혹적 외형의 마녀 |
| Gargoyle | 중~대형 | 방어형 | 날개 달린 석상 악마 |
| Orc | 중~대형 | 야만 강타형 | 거대하고 야만적인 전사 |

### Undead (죽어서 되살아난 존재)

| 베이스 | 크기 | 역할 | 외형 특징 |
|--------|------|------|----------|
| Skeleton | 소~중형 | 밸런스형 | 무장한 해골 전사·궁수 |
| Zombie | 중~대형 | 느린 고체력 | 썩고 무거운 시체 |
| Ghost | 소~중형 | 물리방어↓ 마법저항↑ | 반투명한 원혼 |
| Vampire | 중~대형 | 흡혈 공격형 | 귀족적 외형의 흡혈귀 |
| Lich | 중~대형 | 마법 극대화 | 마법에 집착한 불사 마법사 |

## 몬스터 데이터 구조

| 레이어 | 컬럼 | 역할 |
|--------|------|------|
| monster_type | Normal / Demon / Undead | 헌터 아이템 대상 분류 |
| monster_base | Wolf / Yeti / Troll... | 장비 부위 드롭 분포 결정 |
| monster_grade | 1~4 | 드롭 테이블 품질 결정 |

- **monster_type** → 공통 접사 옵션 `vs_[type]_damage` 대상
- **monster_base** → `monster_drop_equipment.csv`에서 부위별 드롭 가중치 참조
- **monster_grade** → `monster_drop_config.csv`에서 드롭 종류별 확률 참조
- 베이스당 3마리(약/중/강) → **총 48마리** (`fastapi/meta_data/monster_info.csv`)
