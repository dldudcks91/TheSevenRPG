# Game Design 기획 가이드

## 기획 논의 원칙

### 절대 규칙
- 분석과 제안만 한다
- "CSV에 반영할까요?", "문서를 수정할까요?" 같은 마무리 질문 금지
- 문서/데이터 수정은 사용자가 명시적으로 요청할 때만 진행

### 논의 접근법
1. 현재 기획 데이터 확인 (CSV + 마크다운)
2. 수치적 근거 기반 분석 (공식 시뮬레이션, 밸런스 시트)
3. 장단점 비교 제시
4. 최종 결정은 사용자에게

## 게임 핵심 루프

```
스테이지 진입 → 웨이브 전투 (자동) → 몬스터 처치
    ↓                                    ↓
스탯 강화 ← 장비 장착 ← 아이템 드롭 ← 스코어 판정
```

### 스테이지 구조
- 7챕터 × 3스테이지 + 챕터 보스 = 총 28개 스테이지
- 각 스테이지: 다수 웨이브 → 웨이브당 몬스터 스폰

### 전투 시스템
- 1:1 자동 전투 (서버 시뮬레이션)
- 카드 스킬 트리거: "타격 시 n%" (명중 필요) / "공격 시 n%" (명중 무관)
- 상태이상 7종 (챕터별 테마 연계)

## 확정 공식

### 명중 판정
```
hit_rate = clamp(Acc/(Acc+Eva) + LvDiff×0.01 + K, 0.05, 0.95)
K = 0.1 (기본 보정)
```
- 동레벨 동스탯: 60% 명중
- 레벨 1차이: ±1% 명중률

### 드롭 판정
- 스코어 기반 레어도 판정 (ItemDropManager)
- 드롭 테이블: CSV 메타데이터 참조

## 메타데이터 체계

### CSV 파일 (fastapi/meta_data/)
- 장비 데이터, 몬스터 데이터, 드롭 테이블, 스테이지 구성
- 서버 기동 시 `GameDataManager`가 메모리로 로드
- 런타임 변경 불가 (read-only)

### 기획 마크다운 (fastapi/docs/game_design/)
- `GAME_DESIGN.md`: 핵심 기획서
- `story/story_guide.md`: 스토리 가이드
- `story/story_line.md`: 스토리 라인
- `story/chapters/`: 챕터별 상세 스토리

### 데이터 동기화
- 구글 시트 ↔ 로컬 CSV 양방향 동기화
- 다운로드: `sync_data_from_gs.py`, `sync_design_from_gs.py`
- 업로드: `sync_data_to_gs.py`, `sync_design_to_gs.py`

## 밸런싱 논의 프레임워크

### 장비 밸런싱
- 무기 4종: 검, 양손검, 도끼, 창 → 각각 다른 스탯 특화
- 방어구 4종: 갑옷, 투구, 장갑, 신발
- 베이스 스탯 설계 시 레벨 구간별 기대 수치 시뮬레이션

### 몬스터 밸런싱
- 챕터/스테이지별 몬스터 스탯 곡선
- 보스 난이도: 해당 챕터 평균 대비 1.5~2배
- 드롭률과 파밍 효율 상관관계

### 밸런스 검증 방법
```
1. 스프레드시트 시뮬레이션 (공식 대입)
2. 기대 킬타임 계산 (레벨별)
3. 장비 획득 기대치 (파밍 시간 환산)
4. 극단값 체크 (최저/최고 스탯 조합)
```
